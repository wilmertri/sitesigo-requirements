# ADR-001: Python + FastAPI como stack backend

## Fecha
2025

## Estado
Aceptado

## Contexto
Se necesita construir una app de gestion de requerimientos
independiente de SITESIGO (que usa PHP/Laravel).
El desarrollador principal (Fabian) conoce PHP/Laravel
pero el objetivo incluye aprender el ecosistema de
agentes de IA, donde Python es el lenguaje nativo.

## Decision
Usar Python 3.13 + FastAPI como stack backend.

## Razones
- Python es el lenguaje nativo del ecosistema de IA
  (Anthropic SDK, LangChain, behave para BDD)
- FastAPI genera documentacion OpenAPI automaticamente
- Pydantic v2 hace cumplir las reglas de negocio
  en la capa de entrada sin codigo adicional
- pytest es el framework de tests mas expresivo
  para TDD en Python
- Oportunidad de aprender un stack nuevo
  manteniendo los principios conocidos (TDD, Clean Arch)

## Consecuencias
- Curva de aprendizaje inicial en Python/FastAPI
- El frontend puede ser cualquier cliente
  que consuma la API REST
- SQLAlchemy permite cambiar de SQLite (dev)
  a PostgreSQL (prod) sin cambiar el codigo
