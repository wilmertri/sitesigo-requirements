# ADR-002: Enums para Prioridad y TipoRequerimiento

## Fecha
2026

## Estado
Aceptado

## Contexto
Los campos prioridad y tipo de un requerimiento
tienen un conjunto cerrado y conocido de valores
(RN-06). Se debia decidir como representarlos:
strings libres, constantes, o enums.

## Decision
Usar Python Enum heredando de str para ambos campos.

## Razones
- Pydantic valida automaticamente contra los valores
  del Enum sin necesidad de validators manuales
- El IDE autocompletado muestra los valores validos
- Imposible crear un requerimiento con valor invalido
  en tiempo de compilacion, no solo en runtime
- Serializa como string en JSON automaticamente
  por heredar de str

## Alternativas descartadas
- Strings libres: sin validacion, errores en runtime
- Constantes: no hay validacion automatica de Pydantic
- Base de datos de configuracion: sobreingenieria
  para un conjunto de valores que no cambia

## Consecuencias
- Agregar un nuevo tipo requiere cambiar el Enum
  y hacer deploy (cambio controlado y explicito)
- Los tests usan TipoRequerimiento.bug en lugar
  de la cadena "Bug" (mas seguro y expresivo)
