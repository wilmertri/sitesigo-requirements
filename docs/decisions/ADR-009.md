# ADR-009: Alembic para migraciones de base de datos

## Fecha
2025

## Estado
Aceptado

## Contexto
Base.metadata.create_all() solo crea tablas nuevas.
No puede agregar columnas a tablas existentes.
Al agregar proyecto_id a requerimientos en produccion
fue necesaria una migracion manual via SQL.

## Decision
Usar Alembic para todas las migraciones futuras.

## Implementacion
alembic revision --autogenerate -m "descripcion"
alembic upgrade head

Railway ejecuta alembic upgrade head
automaticamente en cada deploy via start.sh.

## Consecuencias
- Migraciones automaticas en cada deploy
- Historial completo de cambios al esquema
- Sin intervenciones manuales en produccion
- Compatible con SQLite (dev) y PostgreSQL (prod)
