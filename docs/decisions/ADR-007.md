# ADR-007: Step matcher regex para Gherkin con strings vacios

## Fecha
2025

## Estado
Aceptado

## Contexto
behave ofrece dos modos de captura en steps:
- parse (default): captura parametros con {nombre}
- re: captura con expresiones regulares

Al implementar el escenario de descripcion vacia
el step matcher parse no captura strings vacios ("").
El patron {descripcion} con parse falla cuando
el valor es una cadena vacia.

## Decision
Usar use_step_matcher("re") con patrones [^"]*
para capturar cualquier string incluyendo vacio.

## Implementacion
from behave import use_step_matcher
use_step_matcher("re")

@when('creo un requerimiento con titulo "(?P<titulo>[^"]*)"')
def step_crear_requerimiento(context, titulo):
    ...

## Consecuencias
- Todos los escenarios con strings vacios funcionan
- Los patrones regex son mas expresivos pero
  mas verbosos que los de parse
- Consistencia: todos los steps usan re
