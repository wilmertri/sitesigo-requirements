# ADR-008: Serializacion de errores Pydantic v2 en routers

## Fecha
2025

## Estado
Aceptado

## Contexto
Pydantic v2 incluye en e.errors() el campo ctx
con el objeto ValueError original, que no es
JSON serializable. Al intentar devolver ese dict
directamente en un HTTPException, FastAPI lanza
un error interno de serializacion.

## Decision
Usar e.json() para serializar el error y luego
json.loads() para convertirlo a dict limpio.

## Implementacion
except ValidationError as e:
    raise HTTPException(
        status_code=422,
        detail=json.loads(e.json())
    )

## Razon
e.json() serializa correctamente todos los campos
incluyendo las excepciones anidadas en ctx.
json.loads() lo convierte a dict limpio que
FastAPI puede serializar sin errores.

## Consecuencias
- Respuestas 422 siempre bien formadas
- Compatible con Pydantic v2 y versiones futuras
- Patron estandar para manejo de ValidationError
  en FastAPI con Pydantic v2
