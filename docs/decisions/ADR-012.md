# ADR-011: Calculo de dias habiles con libreria holidays

## Estado
Aceptado — 2026-06-08

## Contexto
RN-22 requiere calcular dias_habiles entre fecha_inicio
y fecha_final, excluyendo domingos y festivos colombianos.

El campo dias_habiles no se persiste en
RequerimientoValorCampo; se calcula on-the-fly cada
vez que se consulta un requerimiento.

Opciones evaluadas:
A) Libreria Python holidays (colombian holidays incluidos).
B) Tabla festivos en la BD, mantenida manualmente.
C) API externa de calendario colombiano.

## Decision
Usar la libreria holidays de Python.

Instalacion: pip install holidays
Uso basico:
  import holidays
  co_holidays = holidays.Colombia(years=año)
  festivo = fecha in co_holidays

El calculo de dias_habiles vive exclusivamente en
app/services/dias_habiles_service.py con una funcion
pura calcular_dias_habiles(fecha_inicio, fecha_final).

La funcion excluye: sabados, domingos y festivos
colombianos segun la libreria. El rango es inclusivo
en fecha_inicio y exclusivo en fecha_final (convencion
estandar de rangos de fecha).

dias_habiles nunca se guarda en BD. El campo tipo
"calculado" en ProyectoConfigCampo es una bandera
que le indica al frontend que el valor viene del
endpoint de calculo, no de RequerimientoValorCampo.

## Consecuencias
Positivas:
- Sin dependencia de red ni de datos manuales.
- La libreria cubre festivos fijos y moviles de Colombia
  (incluye Semana Santa, festivos de puente, etc.).
- Facil de actualizar con pip.
- La funcion es pura: testeable sin BD ni mocks.

A tener en cuenta:
- La libreria holidays debe agregarse a requirements.txt.
- Si Colombia cambia un festivo por decreto (poco frecuente),
  la actualizacion viene con el siguiente release de holidays.
- El endpoint que devuelve el detalle del requerimiento debe
  calcular y adjuntar dias_habiles cuando el proyecto tiene
  ese campo configurado y ambas fechas estan presentes.

## Alternativas descartadas
- Tabla festivos en BD: descartado por el costo de
  mantenimiento manual y el riesgo de datos desactualizados.
- API externa: descartado por dependencia de red, latencia
  adicional y posible indisponibilidad del servicio.
